# Football Tournament System Frontend

Фронтенд учебного проекта **«Система учёта футбольных турниров»** по дисциплине **«Системная и программная инженерия»**.

Проект реализован на **Next.js + React + TypeScript** и покрывает ключевые сценарии клиентской части:
- авторизация и роли пользователей
- кабинет пользователя
- управление турнирами
- работа с матчами и результатами
- турнирная таблица
- команды, игроки и заявки

## Технологии

- `Next.js 15`
- `React 19`
- `TypeScript`
- `App Router`
- локальный `mock API` для демонстрации интерфейсов без backend

## Что уже реализовано

- Демо-авторизация с ролями:
  `admin`, `organizer`, `referee`, `coach`, `fan`
- Ограничение доступа к разделам по ролям
- Дашборд с ролевыми панелями и быстрыми действиями
- Форма создания турнира с клиентской валидацией
- Форма ввода результата матча с клиентской валидацией
- Форма подачи заявки команды с клиентской валидацией
- Таблицы с поиском, фильтрами и сортировкой
- Mock API-слой для хранения состояния на клиенте

## Основные сценарии

### Организатор
- создаёт турниры
- просматривает и подтверждает результаты матчей
- рассматривает заявки команд

### Судья
- просматривает свои матчи
- вводит результаты и события матча

### Тренер
- отслеживает статус заявки
- подаёт заявку команды на турнир
- просматривает статистику игроков

### Администратор
- управляет ролями пользователей

### Болельщик
- просматривает турнирную таблицу
- следит за открытыми турнирами и статистикой

## Важное замечание

Сейчас проект использует **mock API и локальные данные**, поэтому:
- данные не сохраняются в реальную базу
- авторизация не подключена к backend
- интерфейсы подготовлены как основа под дальнейшую интеграцию
